# 📋 Отчет о внесенных изменениях

**Дата:** 2026-03-10
**Проект:** Tarot Mini App
**Версия:** Protocol v5.6.0

---

## 🎯 Общая цель

Реализация системы онлайн-статуса тарологов с WebSocket heartbeat, улучшенной админ-панелью и отдельным Mini App для тарологов.

---

## ✅ Основные изменения

### 1. Система онлайн-статуса тарологов

**Файлы:** `server/db.js`, `server/server.js`

**Что сделано:**
- Добавлены колонки в БД: `is_active`, `last_heartbeat_at`, `last_ready_at`, `ready_until`, `last_ws_ping`
- Реализована комбинированная логика определения онлайн-статуса:
  1. WebSocket heartbeat (приоритетный, 2 минуты)
  2. HTTP heartbeat (для обратной совместимости)
  3. Ручное подтверждение "Готов консультировать" (30 минут)
  4. Активная сессия чата
- Метод `wsHeartbeat()` для обновления WebSocket статуса
- Метод `setReady()` для ручного подтверждения готовности
- Метод `isRealOnline()` с проверкой всех условий
- Миграции БД автоматические

**API endpoints:**
- `POST /api/tarologist/:id/heartbeat` - обновление heartbeat
- `POST /api/tarologist/:id/ready` - подтверждение готовности
- `GET /api/tarologist/:id/status` - получение статуса

---

### 2. WebSocket Heartbeat

**Файлы:** `server/server.js`

**Что сделано:**
- Реализован WebSocket heartbeat через Socket.IO
- Событие `tarologist-connect` - подключение таролога
- Событие `tarologist-ping/pong` - heartbeat каждые 2 минуты
- Событие `tarologist-status` - ответ с текущим статусом
- 5-минутный таймаут на переподключение
- Обработка disconnect

**Преимущества:**
- Меньше HTTP-запросов (каждые 2 минуты вместо 30 секунд)
- Реальное состояние соединения
- Меньше нагрузки на батарею

---

### 3. Бот для тарологов

**Файлы:** `server/admin-bot.js`

**Что сделано:**
- Команда `/ready` - быстрая активация онлайн-статуса
- Команда `/status` - проверка текущего статуса
- Кнопка **"🟢 Готов консультировать (30 мин)"**
- Меню для админа-таролога (комбинированное)
- Обработка successful_payment через webhook
- Обработка pre_checkout_query
- Добавлена кнопка **"💬 Мои чаты"** - открывает Mini App для чатов

**Роли пользователей:**
- Админ + Таролог → специальное меню с обеими ролями
- Только таролог → меню таролога
- Только админ → админ-меню

---

### 4. Админ-панель

**Файлы:** `public/admin-app/admin.js`, `public/css/admin.css`

**Что сделано:**
- Тарологи НЕ пропадают из списка при отключении
- Визуальное отличие отключенных тарологов (серый цвет)
- Кнопка **"🔕 Отключить"** для активных
- Кнопка **"✅ Подключить"** для отключенных
- Статус "🔴 Отключен" в списке
- Сортировка: активные первые

---

### 5. Отдельный Mini App для тарологов

**Файлы:** `public/tarologist-chat.html`

**Что сделано:**
- Упрощенный интерфейс только для чатов
- Список активных консультаций
- WebSocket чат с клиентами
- Таймер сессии (25 минут)
- История сообщений
- Автоматическое обновление списка чатов

**API endpoints:**
- `GET /api/tarologist/me` - данные таролога
- `GET /api/tarologist/sessions/active` - активные сессии
- `GET /api/chat/session/:id/messages` - сообщения сессии

**Метод БД:** `ChatSession.getActiveByTarologist()`

### 6. Обработка платежей

**Файлы:** `server/server.js`

**Что сделано:**
- Webhook обработка платежей (`/api/bot/webhook`)
- Обработка `successful_payment`
- Обработка `pre_checkout_query`
- Автоматическое создание сессии чата после оплаты
- Уведомление таролога о новой консультации
- Проверка идемпотентности платежей

**Улучшено:**
- Расклад теперь приходит с описаниями карт
- Уведомление таролога содержит инструкцию по ответу (через Mini App)
