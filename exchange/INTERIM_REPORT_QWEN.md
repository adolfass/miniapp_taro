## Что было сделано

### 1. Исправлен баг с онлайн-статусом D P ✅
**Проблема:** Таролог D P постоянно отображался онлайн
**Причина:** 3 зависшие сессии в БД держали статус через `isRealOnline()`
**Решение:**
- Очищены зависшие сессии из БД
- Обновлена логика `isRealOnline()` в `db.js:526-533` - теперь проверяется время сессии (не старше 25 минут)

```javascript
// Теперь учитываются только сессии младше 25 минут
AND datetime(start_time, '+25 minutes') > datetime('now')
```

### 2. Упрощена админка ✅
**Убрано:**
- Кнопка "Отметить выплату" из модального окна таролога
- Модальное окно подтверждения выплаты
- Связанные функции в admin.js

**Причина:** Дублирование функционала - выплаты управляются через вкладку "Выплаты"

### 3. Система оценки консультаций ✅

**Жизненный цикл сессии:**
1. Оплата → создание сессии (`active=1`)
2. 25 минут чата
3. Автоматическое закрытие (`active=0`, `completed=1`)
4. Показ модального окна оценки (через 3 сек)
5. Пользователь оценивает ИЛИ нажимает "Пропустить"

**Упрощенная логика оценки:**
- Оценка доступна только сразу после закрытия сессии
- Нет временных рамок (24 часа убраны)
- Если пропустил - больше нельзя оценить
- Сессия помечается `rated=0` в статистике

**API Endpoints:**
- `GET /api/session/:id/status` - статус сессии
- `POST /api/session/:id/rate` - сохранить оценку
- `GET /api/admin/tarologist/:id/unrated-sessions` - список сессий без оценки

### 4. Статистика без оценки в админке ✅

**В модальном окне таролога (секция "Тарологи") добавлено:**
- Счетчик "Без оценки: X"
- Список сессий с ID клиента и датой
- Данные загружаются асинхронно при открытии модала

**Пример:**
```
Без оценки: 3
Клиент #123    | 11.03.2026, 14:30
Клиент #456    | 11.03.2026, 12:15
Клиент #789    | 10.03.2026, 18:45
```

### 5. Миграция БД ✅

**Добавлены колонки в `chat_sessions`:**
- `rated BOOLEAN DEFAULT 0` - оценена ли сессия
- `rating INTEGER` - оценка (1-5)
- `rating_comment TEXT` - комментарий
- `rated_at DATETIME` - время оценки
