## 📈 Доступные Метрики

### 1. DAU/WAU/MAU (Активные пользователи)

**DAU** — Daily Active Users (активные пользователи за день)  
**WAU** — Weekly Active Users (активные пользователи за неделю)  
**MAU** — Monthly Active Users (активные пользователи за месяц)

**Endpoints:**
```
GET /api/admin/stats/dau?days=30
GET /api/admin/stats/wau?weeks=12
GET /api/admin/stats/mau?months=12
```

**Что показывает:** Популярность и частоту использования приложения

**Интерпретация:**
- DAU/MAU > 0.2 — Отличная вовлечённость (пользователи возвращаются)
- DAU/MAU 0.1-0.2 — Хорошая вовлечённость
- DAU/MAU < 0.1 — Нужно улучшать retention

---

### 2. Воронка (Funnel)

**Описание:** Отслеживание шагов пользователей от входа до оплаты

**Endpoint:**
```
GET /api/admin/stats/funnel?days=30
```

**Шаги воронки:**
1. `app_open` — Открыли приложение
2. `spread_selected` — Выбрали расклад
3. `cards_flipped` — Перевернули карты
4. `payment_completed` — Совершили оплату

**Конверсии:**
- app_to_spread: % открывших → выбравших расклад
- spread_to_flip: % выбравших расклад → перевернувших карты
- flip_to_payment: % перевернувших → оплативших
- app_to_payment: % открывших → оплативших (общая конверсия)

**Что показывает:** На каком этапе теряются пользователи

---

### 3. Конверсия (Conversion)

**Описание:** Процент пользователей, выполнивших целевое действие

**Endpoint:**
```
GET /api/admin/stats/conversion?target=payment_completed&days=30
```

**Формула:**
```
Conversion = (Пользователи с целевым действием / Все пользователи) × 100%
```

**Целевые действия:**
- `payment_completed` — Оплата консультации
- `spread_selected` — Выбор расклада
- `cards_flipped` — Переворот карт

**Что показывает:** Эффективность приложения в достижении бизнес-целей

---

### 4. LTV (Lifetime Value)

**Описание:** Средний доход с одного пользователя за всё время

**Endpoint:**
```
GET /api/admin/stats/ltv?days=30
```

**Метрики:**
- **LTV** — Lifetime Value (доход / все пользователи)
- **ARPPU** — Average Revenue Per Paying User (доход / платящие пользователи)
- **Paying Rate** — % платящих пользователей
- **Total Revenue** — Общий доход
- **Paying Users** — Количество платящих пользователей

**Формула:**
```
LTV = Общий доход / Общее количество пользователей
ARPPU = Общий доход / Платящие пользователи
```

**Что показывает:** Сколько в среднем приносит один пользователь

---

### 5. Retention Rate

**Описание:** Процент пользователей, вернувшихся через X дней после первого входа

**Endpoint:**
```
GET /api/admin/stats/retention?cohort_days=30
```

**Метрики:**
- **D1** — % вернувшихся на следующий день
- **D7** — % вернувшихся через 7 дней
- **D30** — % вернувшихся через 30 дней

**Формула:**
```
Retention Dx = (Вернувшиеся на день X / Новые пользователи) × 100%
```

**Интерпретация:**
- D1 > 40% — Отличный показатель
- D7 > 20% — Хороший показатель
- D30 > 10% — Отличный долгосрочный retention

**Что показывает:** Насколько приложение "цепляет" пользователей

---

### 6. Session Length (Длительность сессий)

**Описание:** Сколько времени пользователь проводит в приложении за одну сессию

**Endpoint:**
```
GET /api/admin/stats/session-length?days=30
```

**Метрики:**
- **Avg Duration** — Средняя длительность (минуты)
- **Median Duration** — Медиана (минуты)
- **P90 Duration** — 90-й процентиль (минуты)
- **Total Sessions** — Общее количество сессий
- **Valid Sessions** — Валидные сессии (исключая выбросы)

**Что показывает:** Насколько вовлекающий продукт

**Интерпретация:**
- > 5 минут — Хорошая вовлечённость
- 2-5 минут — Средняя
- < 2 минут — Нужно улучшать UX

---

### 7. Vitality / K-factor

**Описание:** Вирусный коэффициент роста (рост за счёт рекомендаций)

**Endpoint:**
```
GET /api/admin/stats/vitality?days=30
```

**Метрики:**
- **K-factor** — Вирусный коэффициент
- **Invited Users** — Количество приглашённых пользователей
- **Installed from Invite** — Установившие по приглашению
- **Conversion Rate** — Конверсия приглашения в установку (%)

**Формула:**
```
K = (Приглашённые / Текущие пользователи) × Конверсия в установку
```

**Интерпретация:**
- **K > 1** — Вирусный рост! 🚀
- **K 0.5-1** — Хороший органический рост
- **K < 0.5** — Рост только за счёт маркетинга

**Что показывает:** Насколько быстро приложение растёт за счёт рекомендаций

---

### 8. Revenue (Доход по периодам)

**Описание:** Динамика дохода во времени

**Endpoint:**
```
GET /api/admin/stats/revenue?period=monthly&limit=12
```

**Периоды:** `daily`, `weekly`, `monthly`

**Метрики:**
- **Transactions** — Количество транзакций
- **Revenue** — Общий доход (звёзды)
- **Developer Cut** — Доля разработчика
- **Tarologist Cut** — Доля тарологов

**Что показывает:** Динамику бизнеса и сезонность

---

### 9. Top Tarologists

**Описание:** Топ тарологов по доходу и консультациям

**Endpoint:**
```
GET /api/admin/stats/top-tarologists?limit=10
```

**Метрики:**
- **Consultations** — Количество консультаций
- **Total Revenue** — Общий доход
- **Total Earned** — Заработок таролога

**Что показывает:** Самых эффективных тарологов

## 🔧 Техническая Реализация

### Таблица Events

```sql
CREATE TABLE events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    event_type TEXT NOT NULL,
    event_data TEXT,  -- JSON с дополнительными данными
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### Отслеживаемые события:

- `app_open` — Открытие приложения
- `spread_selected` — Выбор расклада
- `cards_flipped` — Переворот карт
- `payment_completed` — Успешная оплата
- `user_invited` — Приглашение пользователя (для K-factor)
- `app_close` — Закрытие приложения (для Session Length)

## 🎯 Действия на основе метрик

### Если DAU падает:
1. Проверить Retention (возвращаются ли пользователи?)
2. Увеличить маркетинг/рекламу
3. Добавить push-уведомления

### Если Conversion низкая:
1. Анализ воронки — на каком шаге теряем?
2. Упростить процесс оплаты
3. Добавить триггеры ("Только сегодня скидка!")

### Если Retention D1 низкий:
1. Улучшить onboarding
2. Добавить персонализацию
3. Отправлять welcome-расклад

### Если K-factor < 0.5:
1. Добавить реферальную программу
2. Бонусы за приглашения
3. "Пригласи друга — бесплатный расклад"

## 📚 API Reference

### Аутентификация
Все endpoints требуют Telegram initData в заголовке:
```
X-Telegram-Init-Data: <telegram_web_app_init_data>
```

### Параметры

**Периоды:**
- `days` — Количество дней (по умолчанию: 30)
- `weeks` — Количество недель (по умолчанию: 12)
- `months` — Количество месяцев (по умолчанию: 12)

**Дополнительно:**
- `target` — Целевое действие для conversion (по умолчанию: payment_completed)
- `period` — Период для revenue: daily/weekly/monthly (по умолчанию: monthly)
- `limit` — Лимит результатов (по умолчанию: 10-12)
- `cohort_days` — Размер когорты для retention (по умолчанию: 30)
