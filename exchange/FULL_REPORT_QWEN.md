## 💰 ИЗМЕНЁННЫЕ АЛГОРИТМЫ ОПЛАТЫ

### Алгоритм создания инвойса:
```
1. Клиент → POST /api/create-invoice
2. Сервер → Проверка initData
3. Сервер → Создание транзакции в БД
4. Сервер → Запрос к Telegram API (sendInvoice)
5. Сервер → Проверка: есть ли invoice_link в ответе?
6. Сервер → Если да: возврат ссылки клиенту
7. Сервер → Если нет: ошибка "Invoice link not generated"
8. Клиент → tg.openInvoice(url, callback)
9. Клиент → Обработка статуса: paid / cancelled / failed
10. Клиент → Закрытие модального окна при успехе
```

### Алгоритм выплат тарологам:
```
1. Админ открывает "Выплаты" в админке
2. Видит список тарологов с балансом
3. Нажимает "Создать выплату"
4. Вводит сумму в звёздах
5. Статус: pending
6. Админ переводит звёзды вручную через Telegram
7. Нажимает "Отметить выполненной"
8. Статус: completed
9. Баланс таролога обновляется
```
