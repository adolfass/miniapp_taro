## ✅ НОВЫЕ ПРАВИЛА ДЛЯ АГЕНТА

### 1. Агент МОЖЕТ создавать любые .md файлы

**Разрешено:**
- ✅ `REPORT_TO_QWEN.md` — развёрнутый отчёт
- ✅ `CHANGES_SUMMARY.md` — описание изменений
- ✅ `*_GUIDE.md` — инструкции/гайды
- ✅ `*_INSTRUCTION.md` — инструкции для пользователей
- ✅ **ЛЮБЫЕ другие `*.md`** — агент может придумать своё название

### 2. Агент ОБЯЗАН документировать важные изменения

**Если создал новый файл:**
- ✅ Указать название файла в отчёте (`toQwen.md`)
- ✅ Кратко описать что в файле
- ✅ Положить файл в `exchange/`

**Пример:**
```markdown
# В toQwen.md:

## 📁 Дополнительные файлы:

Созданы новые файлы в exchange/:
- `WEBSOCKET_HEARTBEAT_GUIDE.md` — инструкция по WebSocket heartbeat
- `CHANGES_SUMMARY.md` — полный список изменений v5.6.0
```

### 3. Qwen Code ОБЯЗАН читать ВСЕ файлы

**Алгоритм для Qwen:**
```bash
# 1. Получить список ВСЕХ .md файлов
ssh root@server "ls -lt /var/www/tarot-miniapp/exchange/*.md"

# 2. Прочитать ВСЕ файлы (даже с неизвестными названиями)
ssh root@server "cat /var/www/tarot-miniapp/exchange/*.md"

# 3. Прочитать отчёт
git pull origin main
cat exchange/toQwen.md
```

**Проблема которую решили:**

1. Агент создал `WEBSOCKET_HEARTBEAT_GUIDE.md` с инструкцией для фронтенда

**Решение:**
- ✅ Qwen ОБЯЗАН читать ВСЕ .md файлы
