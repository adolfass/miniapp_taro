## 📋 Тест-кейсы

### ✅ Тест 1: Таролог становится онлайн

**Действия:**
1. Таролог открывает бота @goldentarot_bot
2. Нажимает /start
3. Нажимает кнопку "🟢 Готов консультировать (30 мин)"

**Ожидаемый результат:**
- ✅ Приходит подтверждение: "Вы теперь онлайн!"
- ✅ В API статус `is_online: true`
- ✅ Время `ready_until` = текущее + 30 минут

**Проверка:**
```bash
curl -s http://localhost:3001/api/tarologist/7/status | jq '.data.is_online'
# Должно вернуть: true
```

### ✅ Тест 2: Клиент видит таролога онлайн

**Действия:**
1. Клиент открывает Mini App
2. Выбирает расклад (Ежедневный или На ситуацию)
3. Переходит к выбору таролога

**Ожидаемый результат:**
- ✅ Таролог отображается в списке
- ✅ У таролога статус "🟢 Онлайн"
- ✅ Показывается цена консультации

**Проверка:**
```bash
curl -s http://localhost:3001/api/tarologists | jq '.data[] | {name: .name, online: .is_online}'
```

---

### ✅ Тест 3: Оплата консультации

**Действия:**
1. Клиент нажимает "Консультация" у онлайн-таролога
2. Подтверждает оплату (1 звезда в тестовом режиме)
3. Оплачивает через Telegram Stars

**Ожидаемый результат:**
- ✅ Создается транзакция в БД (статус: pending → completed)
- ✅ Создается сессия чата (chat_sessions)
- ✅ Таролог получает уведомление в бота
- ✅ В уведомлении полный расклад с описаниями карт
- ✅ Таролог видит кнопку/ссылку для входа в чат

**Проверка:**
```bash
# Проверить сессии
sqlite3 server/tarot.db "SELECT id, tarologist_id, user_id, active FROM chat_sessions ORDER BY id DESC LIMIT 3;"

# Проверить транзакции
sqlite3 server/tarot.db "SELECT id, status, stars_amount FROM transactions ORDER BY id DESC LIMIT 3;"
```

### ✅ Тест 4: Таролог открывает чат

**Действия:**
1. Таролог получает уведомление о новой консультации
2. Нажимает кнопку "💬 Мои чаты" в боте
3. Открывается Mini App tarologist-chat.html
4. Видит список активных консультаций

**Ожидаемый результат:**
- ✅ Открывается `tarologist-chat.html`
- ✅ В списке видна новая консультация
- ✅ Показывается клиент (ID или имя)
- ✅ Есть таймер сессии (25 минут)
- ✅ WebSocket подключается (статус "Подключено")

**Проверка:**
```bash
# Проверить WebSocket статус
tail -50 /tmp/server.log | grep -E "(WebSocket|socket|connected)"
```

### ✅ Тест 5: Общение в чате

**Действия:**
1. Таролог нажимает на сессию с клиентом
2. Открывается чат
3. Таролог пишет сообщение: "Здравствуйте! Готов ответить на ваши вопросы."
4. Клиент видит сообщение в своём Mini App
5. Клиент отвечает

**Ожидаемый результат:**
- ✅ Сообщение таролога появляется в чате клиента в реальном времени
- ✅ Сообщение клиента появляется в чате таролога в реальном времени
- ✅ Все сообщения сохраняются в БД (таблица messages)
- ✅ Временные метки корректные

**Проверка:**
```bash
# Проверить сообщения
sqlite3 server/tarot.db "SELECT id, sender_type, text, created_at FROM messages WHERE session_id = <ID> ORDER BY created_at DESC LIMIT 5;"
```

### ✅ Тест 6: Таймаут сессии (25 минут)

**Действия:**
1. Ждём 25 минут (или сократить для теста)
2. По истечении времени сессия должна автоматически завершиться

**Ожидаемый результат:**
- ✅ Таймер показывает "00:00" или "Время истекло"
- ✅ Сессия помечается как completed
- ✅ Поле active = 0
- ✅ Чат блокируется (нельзя отправить новые сообщения)

**Проверка:**
```bash
sqlite3 server/tarot.db "SELECT id, active, completed, end_time FROM chat_sessions WHERE id = <ID>;"
```

### ✅ Тест 7: Админ отключает/включает таролога

**Действия:**
1. Админ открывает админ-панель
2. Находит таролога в списке
3. Нажимает "🔕 Отключить"
4. Таролог исчезает из списка для клиентов
5. Админ нажимает "✅ Подключить"

**Ожидаемый результат:**
- ✅ Таролог остаётся в списке админа
- ✅ Статус меняется на "🔴 Отключен"
- ✅ Клиенты НЕ видят отключенного таролога
- ✅ После включения таролог снова виден клиентам

**Проверка:**
```bash
sqlite3 server/tarot.db "SELECT id, name, is_active FROM tarologists;"
```
