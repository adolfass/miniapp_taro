## 🆕 НОВОЕ ПРАВИЛО (добавлено 2026-03-10):

### ⏸️ ПРЕДВАРИТЕЛЬНАЯ ПРОВЕРКА ПЕРЕД ПУШЕМ:

**Перед пушем кода на GitHub OpenCode ОБЯЗАН:**

1. **Показать изменения менеджеру:**
   - Список изменённых файлов
   - Краткое описание внесённых изменений
   - Вопрос: "Нужно ли что-то исправить или добавить?"

2. **Дождаться подтверждения:**
   - ✅ "Отправляй" / "Push it" - можно пушить
   - 🔄 "Подожди" / "Wait" - нужно внести правки
   - ❌ "Не отправляй" / "Don't push" - отменить изменения

3. **Только после подтверждения выполнять `git push`**

**Цель:** Дать менеджеру возможность внести микроправки перед финальным коммитом.
