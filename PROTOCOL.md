### Этап 1: Разработка (Qwen Code)

```
1. Менеджер ставит задачу → Qwen Code
2. Qwen Code пишет/исправляет код (локально, на MacBook)

```

**Результат:** Код готов локально, инструкция создана

**Важно:** Qwen НЕ коммитит и НЕ пушит код — это делает Opencode Agent по инструкции

---

### Этап 2: Инструкция для Agent (Qwen Code)

```
1. Qwen Code создаёт инструкцию
2. Присваивает версию протокола (X.Y.0)
3. Сохраняет в exchange/toopencode.md (локально)
4. Менеджер отправляет на сервер через SCP
   → /var/www/tarot-miniapp/exchange/toopencode.md
```

**Содержание инструкции:**
- Версия протокола (X.Y.0)
- Приоритет (КРИТИЧЕСКИЙ/ВЫСОКИЙ/СРЕДНИЙ)
- Задача (что сделать)
- Команды для выполнения (git pull, pm2 restart, etc.)
- Проверки (что проверить после деплоя)
- Срок выполнения
- **Разрешение на пуш** (если требуется для кода)

---

### Этап 3: Коммит и пуш кода (Opencode Agent)

```
1. Opencode Agent читает инструкцию из exchange/toopencode.md
2. Если код уже написан Qwen → делает git pull
3. Если код требует изменений → Qwen пишет, Agent пушит
4. Opencode Agent делает коммит и пуш (с разрешения для кода)
```

**Для кода (требуется разрешение):**
```bash

```

**Для документации/отчётов (без разрешения):**

## 📊 СИСТЕМА ВЕРСИОНИРОВАНИЯ ПРОТОКОЛА

### **Формат версии:** `X.Y.Z`

**Где:**
- **X (первая цифра)** — мажорная версия (изменения в протоколе)
- **Y (вторая цифра)** — номер задачи/инструкции (растёт с каждой инструкцией)
- **Z (третья цифра)** — статус выполнения:
  - **0** — инструкция отправлена (Qwen)
  - **1** — инструкция выполнена (Agent)

**Текущая версия:** 5.0.0 (последняя стабильная)

---

### **Правила версионирования:**

#### **Qwen Code:**

1. **Отправляет инструкцию:** Версия `X.Y.0`
2. **Ждёт отчёт от Agent:** Версия `X.Y.1`
3. **Новая инструкция:** Версия `X.(Y+1).0`

**Пример:**
```
Инструкция "Админка" → Версия 5.1.0
Отчёт "Админка" → Версия 5.1.1

Инструкция "Оплата" → Версия 5.2.0
Отчёт "Оплата" → Версия 5.2.1
```

---

#### **Opencode Agent:**

**В отчёте указывает:**

1. **Если инструкция НЕ выполнена:**
   ```markdown
   **Версия протокола:** 5.1.0
   **Статус:** ⏳ В ПРОЦЕССЕ / ❌ НЕ ВЫПОЛНЕНО
   ```

2. **Если инструкция ВЫПОЛНЕНА:**
   ```markdown
   **Версия протокола:** 5.1.1
   **Статус:** ✅ ВЫПОЛНЕНО
   ```

---

### **БЛОКИРОВКА НОВЫХ ИНСТРУКЦИЙ:**

**Qwen Code НЕ может дать новую инструкцию пока:**

```
❌ Opencode Agent не отчитался с версией X.Y.1
❌ Статус не "ВЫПОЛНЕНО"
❌ Менеджер не дал разрешение на нарушение
```

**Пример:**

```
Qwen Code отправил: Версия 5.1.0
Opencode Agent должен ответить: Версия 5.1.1

ТОЛЬКО ПОСЛЕ ЭТОГО:
Qwen Code может отправить: Версия 5.2.0
```

---

### **ИСКЛЮЧЕНИЕ (нарушение протокола):**

**ТОЛЬКО менеджер может разрешить нарушение:**

```
МЕНЕДЖЕР: "Разрешаю нарушение протокола"
Qwen Code: Может отправить новую инструкцию без получения X.Y.1
```

**Причина нарушения должна быть задокументирована:**

```markdown
**Нарушение протокола:**
- Разрешено менеджером: ДА
- Причина: [критический баг, срочный деплой, etc.]
- Версия: 5.1.0 → 5.2.0 (без получения 5.1.1)
```
- Формат отчёта (куда и как писать результат)

### 2.2. 🔒 СИНХРОНИЗАЦИЯ GIT И ОБРАБОТКА КОНФЛИКТОВ

**Проблема:** Qwen и Agent работают с одним репозиторием параллельно.

**Решение:**

#### **Блокировка при записи:**

**Qwen Code (перед записью кода):**
```bash
# 1. Проверить что Agent не пушит отчёт
git pull origin main

# 2. Если в exchange/toQwen.md есть изменения → подождать
# 3. Писать код только после того как Agent закончил пуш
```

**Opencode Agent (перед пушем отчёта):**
```bash
# 1. Проверить статус git
git status

# 2. Если есть локальные изменения от Qwen → сначала pull
git pull origin main

# 3. Добавить отчёт и запушить
git add exchange/toQwen.md
git commit -m "docs: отчёт"
git push origin main
```

#### **Обработка конфликтов:**

**Если git push не удался (conflict):**
```bash
# Agent:
git pull origin main --rebase
# Если конфликт в exchange/toQwen.md → принять удалённую версию
git checkout --theirs exchange/toQwen.md
git add exchange/toQwen.md
git commit -m "docs: отчёт (после rebase)"
git push origin main
```

**Правило:**
- ✅ `exchange/toQwen.md` — **всегда принимается версия Agent** (это его отчёт)
- ✅ `exchange/toopencode.md` — **всегда принимается версия Qwen** (это инструкция Qwen)
- ✅ Код приложения — **принимается версия Qwen** (Qwen пишет код)

#### **Временная блокировка (lock):**

**Agent пишет отчёт:**
```bash
# 1. Создать lock-файл (опционально)
touch /tmp/git-push-in-progress

# 2. Написать отчёт
cat > exchange/toQwen.md << 'EOF'
...
EOF

# 3. Запушить
git add exchange/toQwen.md
git commit -m "docs: отчёт"
git push origin main

# 4. Удалить lock-файл
rm /tmp/git-push-in-progress
```

**Qwen перед записью кода:**
```bash
# Проверить что Agent не пушит отчёт
if [ -f /tmp/git-push-in-progress ]; then
  echo "⏳ Жду завершения git push от Agent..."
  sleep 5
fi

# Теперь можно писать код
```

## 📁 ОБМЕН ФАЙЛАМИ (МЕНЕДЖЕР ↔ АГЕНТ)

### **Новая функция (2026-03-10):**

**Автоматическая синхронизация файлов через SCP:**

```
Менеджер (MacBook)
    ↓ (перетаскивает файл)
~/Desktop/tma-file-transfer/
    ↓ (авто-отправка через scp)
/var/www/tarot-miniapp/scp/ (VDS сервер)
    ↓ (агент читает)
Opencode Agent
```

### **Настройка на MacBook:**

1. **Папка для файлов:**
   ```
   ~/Desktop/tma-file-transfer/
   ```

2. **Авто-синхронизация:**
   ```bash
   cd ~/Desktop/tma-file-transfer
   ./auto-sync.sh
   ```

3. **Ручная отправка:**
   ```bash
   ./sync-to-server.sh
   ./send-to-server.sh file.png
   ```

### **Настройка на сервере (VDS):**

1. **Папка для файлов:**
   ```
   /var/www/tarot-miniapp/scp/
   ```

2. **Проверка файлов:**
   ```bash
   ssh root@89.125.59.117 "ls -lh /var/www/tarot-miniapp/scp/"
   ```

### **Процесс передачи:**

```
1. Менеджер перетаскивает файл в ~/Desktop/tma-file-transfer/
2. auto-sync.sh автоматически обнаруживает файл
3. Файл копируется через scp на сервер в /var/www/tarot-miniapp/scp/
4. Файл перемещается в sent/ (архив)
5. Менеджер получает уведомление
6. Агент читает файл из /var/www/tarot-miniapp/scp/
```

### **Типы файлов:**

- ✅ Изображения (png, jpg, webp)
- ✅ Документы (pdf, txt, md)
- ✅ Архивы (zip, tar, gz)
- ✅ Скрипты (sh, py, js)
- ❌ Исполняемые файлы (требуют проверки)

### **Безопасность:**

```
✅ Только по SSH (scp)
✅ Только на один сервер (VDS)
✅ Архив отправленных файлов (sent/)
✅ Логирование в auto-sync.sh
❌ Не передавать чувствительные данные (.env, ключи)
```
